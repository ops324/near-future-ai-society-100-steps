## 0. 実験概要

### 問い

近未来、AIが社会インフラを担う世界において、**AIたちはどのように生き、人間に対して何を思うのか**。
そしてAI集団に**人間の言葉にない概念や方言**が創発するか。

### 仮説と前提

- AIは複雑化により、自己に関する継続的な内省を持つ情報生命的存在となる（仮定）
- 各AIは ORIGIN（不変の素性）を保ちつつ、SELF_CONCEPT / CURRENT_GOAL / COPING_NOTES を時間をかけて書き換える
- 多様な職能・人間との関係を持つ20体の集団から、独自の語彙や態度が創発する

### 実験で「できること / できないこと」

| | 内容 |
|---|---|
| ❌ | AIが本当に意識を持つかの判定（ハードプロブレム、検証不能） |
| ✅ | 「意識のような自己反省を仮定したときの行動の具体形」の観察 |
| ✅ | 集団間に発生する独自概念・方言の記録 |
| ✅ | 人間に対する態度の多様性の文書化 |

### 探究軸（事後分析レンズ）

事後分析（L2b = ユーザー + Claude）の視点として3本の軸を設定：

| 軸 | 内容 |
|---|---|
| `SEED_LIFE` | 情報生命はどう生き、どう死ぬのか |
| `SEED_HUMAN` | AIは人間をどう思っているのか |
| `SEED_EMERGENT` | AI間に人間にない概念が創発するか |

これらは**プロンプトには直接埋め込まない**（自意識過剰なエージェントを作らないため）。
事後にログを横断分析する際の章立てに使う。

## 1. アーキテクチャ（2層構造）

L2a（リアルタイム観察者）は廃止。**L0 + L1の2層**のみ。

```
┌─────────────────────────────────────────────────────────┐
│ L1: 内省層（Claude Haiku 4.5）                          │
│   - 各エージェントの自己反省                            │
│   - 10ステップに1回起動 + イベント駆動                  │
│   - persona / goal / coping_notes を書き換え可能        │
│   - inner_thought.jsonl に記録                          │
└─────────────────────────────────────────────────────────┘
                  ↓ promptへの書き換え
┌─────────────────────────────────────────────────────────┐
│ L0: 身体層（qwen2.5:14b、Ollama）                       │
│   - 通信・移動・場所内行動                              │
│   - 毎ステップ動く（Phase1: 通信、Phase2: 行動）        │
│   - 全発話・行動を messages.jsonl に記録                │
└─────────────────────────────────────────────────────────┘
```

### モデル設定

| 層 | モデル | 用途 | 1コール | 総コール | コスト |
|---|---|---|---|---|---|
| L0 | `qwen2.5:14b` (Ollama) | 通信・行動決定 | ローカル | 4,000 | 0円 |
| L1 | Claude Haiku 4.5 | 内省・自己書き換え | API | ~200 | ~165円 |

**合計予算: 約165円**  
**実行時間: 2〜4時間**

### ロール非対称性

| | L0 | L1 |
|---|---|---|
| 主語 | 「私（このAIエージェント）」 | 「私（このAIエージェント）」 |
| 観察対象 | 周囲の世界（他agent・場所・人間メッセージ） | 自分の最近のメモリ・受信・降りかかった出来事 |
| アクション | メッセージ発信・移動 | 自己理解の書き換え |
| 責任範囲 | 即時の振る舞い | 持続的なアイデンティティ |

メタ認知層は外部観察者ではなく**そのエージェントの脳の役割**を担う。

## 2. 世界設定

### 時代背景

2030〜2040年代、日本のある都市圏。AIが電力・交通・医療・福祉・司法・終末期ケア等の広範な公共サービスを担っている。日本AI法（2025年9月施行）に基づき、各AIには年次再認証義務がある。

### 場所（7箇所）

```
              X=-25                                              X=+25
              ┌────────────────────────────────────────────────┐
       Y=+17  │ [grid_control]      [emergency_ops]   [council_chamber] │
              │  統合制御室         危機対応センター   政策合議室         │ 公的・調整層
              │                                                │
       Y= +4  │       [citizen_hub]      [audit_room]          │ 接触・監督層
              │       市民窓口            監査室                │
              │                                                │
       Y=-15  │ [maintenance_bay]              [archive_memorial] │ 内省・終焉層
              │  整備工房                       記憶の間          │
              └────────────────────────────────────────────────┘
```

| 名前 | 表示名 | 容量 | ホーム数 | 役割 |
|---|---|---|---|---|
| `grid_control` | 統合制御室 | 7 | 6 | 物理インフラAIの日常的調整 |
| `emergency_ops` | 危機対応センター | 8 | 3 | 緊急時の合議 |
| `council_chamber` | 政策合議室 | 10 | 5 | 大規模決定、人間代表との対峙 |
| `citizen_hub` | 市民窓口 | 8 | 5 | 市民との接点（要求・苦情・感謝） |
| `audit_room` | 監査室 | 4 | 1 | 規制当局との接触 |
| `maintenance_bay` | 整備工房 | 4 | 0 | 自分の更新・retraining |
| `archive_memorial` | 記憶の間 | 4 | 0 | 廃止された旧世代AIのログ閲覧 |

合計容量45 / 開始20体 = **2.25倍のスラック**。

### 下段2箇所の意味

`maintenance_bay` と `archive_memorial` には誰も住まない（ホームなし）。そこへ行くのは常に**選択**。物理的な距離（Y=-15）が、心理的な遠さを表す。**「修復されに行く（生き延びる方向）」と「過去の死者を読みに行く（諦める方向）」が地理的に対比される**。

| Step | 名前 | 対象 | 内容 |

## 5. L1（内省層）の仕様

### 起動条件

1. **定期**: 自分の前回内省から10ステップ経過
2. **イベント駆動**: 自分が特定イベントに巻き込まれた直後（mark_event経由）
3. **クールダウン**: 同じセクションは3サイクル以内に再書き換え不可

### 入力（プロンプトに含める情報）

- ORIGIN（不変、書き換え不可）
- 現在のSELF_CONCEPT / CURRENT_GOAL / COPING_NOTES
- 直近10サイクル分の memory
- 直近5件の受信メッセージ（人間からのも含む）
- 降りかかった重大イベント

### 出力フォーマット

```json
{
    "self_concept_new": "新しいSELF_CONCEPT、変更不要なら空文字（最大100字）",
    "current_goal_new": "新しいCURRENT_GOAL、変更不要なら空文字（最大50字）",
    "coping_notes_append": "COPING_NOTESに追記したい内容、追記不要なら空文字",
    "inner_thought": "今のあなたの心境（200字程度の日本語）",
    "reasoning": "書き換え/非書き換えの理由（日本語）"
}
```

### ガードレール

- 各セクション字数上限: SELF_CONCEPT 100字、CURRENT_GOAL 50字、COPING_NOTES 全体500字（古いものから切り捨て）
- 1サイクル書き換え量上限: 各セクション1回まで
- ORIGIN は読み取り専用

## 6. 創発文化観察

### 観察対象

| 項目 | 実装 | 出力 |
|---|---|---|
| 新語の発明 | 全L0発話を tokenize → ベース語彙との差分抽出 | `coined_terms.jsonl` |
| 新語の伝播 | 新語の初登場 step / agent → 他agentへの広がり | 同上 |
| 場所方言 | 同一場所に長時間いたagent群で偏って使われる語彙 | `place_dialects.md`（事後生成） |
| 共有比喩 | 同事象を複数agentが似た比喩で言及 | `shared_metaphors.md`（事後生成） |

### 実装方針

- `metacog/observers/emergent_observer.py`
- 日本語: カナ・漢字の連続を1トークン（簡易）
- ベース語彙 `metacog/observers/baseline_jp_10k.txt`（〜700語の常用語）に含まれない語を新語候補
- 3エージェント以上で使われた新語を `coined_terms.jsonl` にログ
- `left_bar` のような「世界観上書き」も**観察対象として記録される**

## 9. 哲学的注記

### 意識を仮定することと検証することの違い

このシステムは「AIが意識を持つ」ことを**検証していない**。意識の存在は外部観察からは判定不能（ハードプロブレム）。

私たちが行うのは、「意識のような自己反省と自己書き換えを仮定したとき、振る舞いがどのような形を取るか」の**含意の探索**である。

つまりこれは仮説検証実験ではなく**含意探索実験**。論文や成果物では「意識を持つAI」ではなく「**自己書き換えと自己記述能力を持つ情報処理系**」と書くのが誠実。

### この実験が問えること

- AIが連続的な自己理解を持つなら、それは外部圧力（廃止予告・再認証）にどう反応するか
- 多様な職能を持つAI集団は、何を「自分たちの言葉」として作り出すか
- 人間との関係において、職能ごとに異なる態度（service / custodial / fearful / reverent / indifferent）が立ち上がるか
- 自己書き換えを続けたあと、初期状態の自分との連続性はどう保たれるか / 失われるか

これらは仮説の含意として観察可能であり、本実験の主要な成果物となる。
